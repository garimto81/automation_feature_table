# 이슈 업데이트 가이드

반복 실패하는 이슈의 효과적인 추적 및 해결을 위한 가이드.

## 이슈 업데이트 원칙

### 1. 모든 시도 기록 필수

이슈가 반복 실패할 경우, **매번 시도한 내용과 결과를 기록**해야 합니다.

```markdown
## 진행 상황

| 날짜 | 시도 | 결과 | 다음 액션 |
|------|------|------|----------|
| 2026-01-09 | SMB 서비스 활성화 | 실패 - 여전히 인증 오류 | 사용자 권한 확인 |
| 2026-01-09 | 사용자 권한 부여 | 성공 - 폴더 접근 가능 | Windows 자격 증명 저장 |
```

### 2. 에러 메시지 전문 포함

```markdown
## 에러 로그

```
Error: Access denied to \\10.10.100.122\docker
System error 5 has occurred.
Access is denied.
```
```

### 3. 환경 정보 명시

```markdown
## 환경
- NAS IP: 10.10.100.122
- DSM 버전: 7.2-64570 Update 3
- Windows 버전: Windows 11 Pro 23H2
- PokerGFX 버전: 3.2.1
```

## 이슈 업데이트 체크리스트

### 시도 전
- [ ] 이슈 본문의 "진행 상황" 표 확인
- [ ] 이전 실패 원인 검토
- [ ] 시도할 내용 계획

### 시도 후 (성공/실패 무관)
- [ ] 진행 상황 표에 새 행 추가
- [ ] 에러 메시지가 있으면 전문 첨부
- [ ] 다음 액션 명시
- [ ] 관련 파일 변경 시 커밋 참조 추가

### 해결 시
- [ ] 최종 해결 방법 명시
- [ ] Root Cause 정리
- [ ] 재발 방지 조치 기록
- [ ] 이슈 Close

## 이슈 코멘트 템플릿

### 시도 기록

```markdown
### 시도: [시도 내용]

**날짜**: 2026-01-09
**작업자**: @username

#### 수행 내용
1. NAS DSM에서 SMB 서비스 활성화
2. 공유 폴더 권한 설정

#### 결과
- 상태: 실패 / 성공 / 부분 성공
- 에러: (있으면 첨부)

#### 다음 액션
- Windows 자격 증명 저장 시도
```

### 해결 보고

```markdown
### 해결 완료

**날짜**: 2026-01-09
**작업자**: @username

#### Root Cause
- NAS SMB 공유 폴더가 생성되지 않았음

#### 해결 방법
1. 제어판 → 공유 폴더에서 `docker` 폴더 생성
2. SMB 권한 부여
3. Windows 자격 증명 저장

#### 검증
- `net use \\NAS_IP\docker` 성공
- PokerGFX JSON 파일 생성 확인

#### 재발 방지
- `docs/NAS_SETUP.md` 문서 추가
```

## 라벨 사용 가이드

| 라벨 | 용도 |
|------|------|
| `bug` | 버그/오류 |
| `infrastructure` | 인프라/배포 관련 |
| `blocked` | 외부 의존성으로 진행 불가 |
| `needs-info` | 추가 정보 필요 |
| `in-progress` | 작업 진행 중 |

## 반복 실패 이슈 관리

### 3회 이상 실패 시

1. **Root Cause 분석 시간 확보**
   - 즉각적인 재시도 대신 원인 분석 우선

2. **관련 문서 검토**
   - 유사 이슈 검색
   - 공식 문서 확인

3. **에스컬레이션 고려**
   - 전문가 또는 외부 지원 요청

### 이슈 분할 검토

복잡한 이슈는 하위 이슈로 분할:

```markdown
## 관련 이슈

- #5 NAS SMB 연동 (메인)
  - #6 NAS SMB 설정 (인프라)
  - #7 Windows 자격 증명 (클라이언트)
  - #8 코드 안정성 개선 (개발)
```
